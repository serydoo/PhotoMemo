# 配置保存、备份与恢复

MemoMark 会把一个记忆对象的资料、时间锚点、卡片内容、显示方式和输出设置组合成一套完整配置。记忆对象和配置都使用内部 ID 识别，因此重名不会互相覆盖。

## 保存为当前配置

在配置中心点击“保存为当前配置”后，当前页面中的内容与输出设置会一起保存。后续从 Apple Photos 分享或在主程序选择照片时，都使用这套已保存配置。

如果没有调整输出设置，MemoMark 会保留系统推荐的默认输出；调整后再次保存，新设置会覆盖当前配置中的输出部分。

## 保存到本地

“保存到本地”会为当前配置创建一份独立备份。备份不会自动改变正在使用的配置，也不会影响已经开始处理的照片。

删除主程序中的配置不会同时删除本地备份。本地备份需要在本地配置库中单独删除。

## 导入与恢复

导入 `.memomarkconfig` 文件时，MemoMark 会先检查文件版本、完整性和随附资源，检查完成前不会修改当前配置。

- 同名但 ID 不同的记忆对象会分别保留。
- 与现有配置 ID 相同的备份默认恢复为副本，避免覆盖。
- 找不到原相册时会安全回退，由用户重新选择。
- 找不到头像或自定义标识时会保留其余配置，并使用安全的默认显示。
- “恢复”只加入配置库；“恢复并设为当前”才会通过正常保存流程应用到后续任务。

## 正在处理的照片

每个处理任务在开始时都会冻结一份配置快照。之后即使修改、恢复或删除配置，已经开始的任务仍按原快照完成，避免一批照片中途改变样式。

## 分享数量

Share Extension 当前一次最多接收 20 张照片。美好的记忆适合慢慢整理，可以分几次分享，也能让处理过程更稳定。
