# Developer Atlas 施工清单

## 目标

在仓库中建立一套可以长期维护的 `Developer Atlas`，帮助后续开发快速定位：

- 功能应该落在哪一层
- 会影响哪条链路
- 先看哪些文件
- 哪些区域风险最高

## 交付策略

这次不做“一次写完全部 24 章”的大爆发，而是按增量方式分三期交付。

这样可以保证：

- 每一期都已经可读、可用
- 目录结构稳定后再继续深挖
- 不会因为一次性过多结论而失真

- `00_阅读指南`

- `09_代码依赖图`
- `19_开发入口（Developer GPS）`
- `21_风险分析（哪些地方不能乱改）`
- `diagrams/`

### 目标

让开发者第一次进入仓库时，先建立一张“能导航的地图”。

### 验收标准

- 能解释 iOS 入口如何到达 V1 和 Configuration Center
- 能说明 V1 与共享协调层之间的关系
- 能说明 Photo -> Metadata -> Card Build -> Render -> Export 的主链路
- 能指出当前首批高风险文件和原因

## 本轮完成定义

当下面几件事都成立时，本轮算完成：

1. `Docs/DeveloperAtlas/` 已建立
2. 首期核心文档已经可读
3. 图和正文之间已互相链接
4. 读者可以不先翻源代码，就知道：
