# 术语词典（中英文）

## 使用原则时光记当前最重要的语言规则是：

- 用户面尽量使用中文产品语言
- 代码面保留必要英文类型名
- 不把旧工作流词汇重新带回用户概念

## 推荐术语

### Configuration Center

中文：

- 配置中心

含义：

- 长期配置对象的中心，不是日常批量工作台。

相关代码：

- `ConfigurationCenteriOSView`
- `ConfigurationSession`

### Configuration Session

中文：

- 配置会话

含义：

- 当前配置中心内的对象状态和表现状态。

相关代码：

- `ConfigurationSession`

### Memory Subject

中文：

- 记忆对象

含义：

- 照片记忆所围绕的人、关系或生命上下文。

相关代码：

- `MemorySubject`
- `MemorySubjectAdapter`

### Memory Card

中文：

- 记忆卡

含义：

- 时光记展示记忆的核心卡片对象和交互表面。

相关代码：

- `InteractiveMemoryCard`
- `RecordCard`

### Object Inspector

中文：

- 对象检查器

含义：

- 配置中心右侧编辑当前对象的区域。

### Preset

中文：

- 预设

含义：

- 用户面推荐使用“预设”，不要用“模板”作为用户主语言。

代码说明：

- 内部代码仍有 `Template` / `TemplatePreset`
- 用户文案优先用 Preset / 预设

### Template

中文：

- 模板

含义：

- 当前代码层的内部结构名。

使用边界：

- 可以在代码和技术文档里使用
- 面向用户时优先说“预设”

### Time Anchor

中文：

- 时间锚点

含义：

- 用于计算照片拍摄时间与某个重要日期之间的关系。

相关代码：

- `Anchor`
- `AnchorEngine`
- `MemorySubject.TimeAnchor`

### Life Anchor

中文：

- 生命锚点

含义：

- 某个生命事件或长期关系里的重要参考点。

### Memory Workflow

中文：

- 记忆工作流

含义：

- 时光记拥有的产品能力边界。

推荐表达：

```text
Apple Photos -> Share -> MemoMark -> Processing -> Notification -> Apple Photos
```

### Share Intake

中文：

- 分享接单

含义：

- 从系统分享、open URL 或 share extension 接收照片请求的入口过程。

相关代码：

- `ExternalPhotoIntakeCenter`
- `ExternalPhotoIntakeStore`
- `ShareCoordinator`

### Batch Queue

中文：

- 批处理队列

含义：

- 保存和执行照片处理任务的队列状态。

相关代码：

- `BatchQueueStore`
- `QueueRepository`

### Renderer

中文：

- 渲染器

含义：

- 根据已组合好的输入生成最终视觉输出。

使用边界：

- 不应继续成为新布局规则的唯一来源。

## 禁用或谨慎使用术语

### Workspace

中文：

- 工作区

状态：

- 当前用户面不推荐。

说明：

- 旧代码中仍有 `Workspace` 相关命名，属于历史实现语言。

### Import

中文：

- 导入

状态：

- 用户工作流中不推荐作为主概念。

说明：

- 技术内部可以描述图片读取或导入服务，但日常工作流应优先说 Apple Photos 分享进入。

### Dashboard

中文：

- 仪表盘

状态：

- 不推荐。

### Task Center

中文：

- 任务中心

状态：

- 不推荐作为产品主概念。

### Photo Manager

中文：

- 照片管理器

状态：

- 禁止用于定义时光记。

说明：

- Apple Photos 才是照片管理系统。

### EXIF Tool

中文：

- EXIF 工具

状态：

- 禁止用于定义时光记。

说明：

- 时光记使用 EXIF，但不是 EXIF 查看器。
