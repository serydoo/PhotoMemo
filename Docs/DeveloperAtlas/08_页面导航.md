# 页面导航

## 先看真实入口

当前 iOS 导航不是“打开 App 就直接到 V1”。

更真实的路径是：

```text
App entry
-> PhotoMemoiOSHomeView
-> PhotoMemoRootSceneView
-> PhotoMemoiOSTemporaryEntryView
-> V1 Preview 或 Configuration Center
```

这意味着：

- V1 是一个重要入口
- 但不是唯一入口
- 当前导航层本身就是过渡期结构的一部分

## V1 页面结构

[PhotoMemoiOSV1View.swift](/Users/rui/Desktop/PhotoMemo-main-archive-20260702/Source/PhotoMemo/PhotoMemo/iOS/Views/PhotoMemoiOSV1View.swift) 内部主要是四个 tab：

- `home`
- `editor`
- `output`
- `settings`

这四块分别大致对应：

- 首页摘要与状态
- 模块编辑与预览
- 输出方式与相册目标
- 设置与默认配置

## Configuration Center 页面结构

Configuration Center 由 [ConfigurationCenteriOSView.swift](/Users/rui/Desktop/PhotoMemo-main-archive-20260702/Source/PhotoMemo/PhotoMemo/iOS/Views/ConfigurationCenteriOSView.swift) 承载。

它的结构更接近：

```text
Library
-> Interactive Memory Card
-> Object Inspector
```

这对应仓库冻结规则中的 IA-002 架构。

## 为什么这一章重要

很多后续开发问题，本质上不是“不会写代码”，而是没有先分清：

- 这是入口层问题
- 这是 V1 页面问题
- 这是 Configuration Center 问题
- 这是共享运行时问题
