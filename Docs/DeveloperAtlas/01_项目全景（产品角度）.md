# 项目全景（产品角度）

## 它是什么

PhotoMemo 当前的产品身份不是“照片管理器”，也不是“加字水印工具”。

它的主定义是：

- 本地优先 Memory Presentation Engine
- Apple Photos 工作流里的 Memory Capability
- 通过一张新的输出图像表达记忆意义的系统

## 产品核心问题

PhotoMemo 不是在回答“照片怎么保存”，而是在回答：

- 这张照片发生在什么时候
- 这张照片在谁的人生里
- 这张照片在生命时间线上的位置是什么
- 哪些信息值得多年后重新阅读

## 当前产品外形

从产品角度，当前仓库其实存在三层同时并存：

1. `Apple Photos -> Share -> Processing -> Notification -> Apple Photos`
这是长期目标工作流，也是 V2 的主产品姿态。

2. `Configuration Center`
这是长期配置中心，用来编辑对象，不是日常读图工作台。

3. `V1 iPhone surface`
这是当前最具体、最容易直接理解的一层产品表面，仍然承载很多配置、预览、输出和过渡逻辑。

## 当前最重要的产品边界

### Apple Photos 是主照片世界

PhotoMemo 不拥有照片管理。

它不负责：

- 照片库浏览
- 云同步
- 地图
- 相册系统的全量替代

它负责：

- 读取照片事实
- 结合 Memory Subject / Anchor / Preset
- 生成一张新的记忆版本图像
- 在可能的情况下写回系统照片库

### Configuration Center 是配置中心

Configuration Center 的产品责任是：

- 编辑对象
- 配置长期默认行为
- 预览真实 Memory Card

不是：

- 日常批量工作台
- 导入大厅
- Dashboard

### V1 是现阶段最重要的“真实工程入口”

虽然仓库主方向已经是 V2 / IA-003，但从“后续加功能要落哪里”这个现实问题出发，V1 仍然是最有必要先梳理清楚的产品线。
