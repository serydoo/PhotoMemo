# 目录职责

## 总览

当前 `Source/PhotoMemo/PhotoMemo/` 不是一个按单一架构理想形态排布的目录，而是历史产品线、V2 架构边界和 iOS 现实入口共存的结果。

首轮理解时，建议按“职责聚类”看，而不是按 Finder 顺序看。

### `Repositories`

职责：

- Settings、Queue、Diagnostics、PhotoLibrary、Configuration 的访问边界

特点：

- 它们比 Service 更偏“仓库语义”
- 它们比 Coordinator 更靠近持久化与数据来源

### `Renderers`

职责：

- 画最终输出图像

当前提醒：

- 根据仓库规则，Renderer 不应继续吸纳新的布局真相
- 目前它依然重要，但不再是唯一中心

### `Views`

职责：

- 较早期的 macOS / 通用视图结构
- 历史主编辑流 `MainView`
- 组件、预览、模板、锚点等 UI

当前地位：

- 仍有大量历史价值
- 但本轮 V1 Atlas 不把它当作 iPhone 主入口

### `iOS`

职责：

- iOS App
- Share Extension
- iOS Views
- Activity / Live Activity

当前地位：

- V1 主舞台
- iPhone 配置与分享接单现实入口

## 代码量感知

当前目录大致规模：

- `ConfigurationCenter`：42 个 Swift 文件
- `Views`：44 个 Swift 文件
- `iOS`：70 个 Swift 文件
- `Services`：20 个 Swift 文件
- `Models`：25 个 Swift 文件

这说明：

- 当前仓库的主要复杂度已经不在单个 renderer 文件
- 而是在 iOS surface、Configuration Center、共享服务和状态边界的交汇处
