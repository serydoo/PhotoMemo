# 后续深化路线

## 目的

Developer Atlas 第一版已经建立了导航、流程、分层、索引和维护规则。

后续深化不应该继续横向铺目录，而应该按高价值主题逐步下钻。

## 路线 1：V1 主界面精读

### 目标

把 [PhotoMemoiOSV1View.swift](/Users/rui/Desktop/PhotoMemo-main-archive-20260702/Source/PhotoMemo/PhotoMemo/iOS/Views/PhotoMemoiOSV1View.swift) 从“知道它很重要”推进到“知道每个主要 section 负责什么”。

### 建议输出

- V1 tab 结构图
- V1 本地状态表
- V1 方法分组表
- V1 与 `V1*Coordinator` 的关系表

### 触发时机

准备继续修改 V1 页面或抽取 V1 子组件前。

## 路线 2：Settings / Snapshot 深挖

### 目标

搞清：

- 哪些设置是页面态
- 哪些设置会持久化
- 哪些设置进入 `BatchConfigurationSnapshot`
- 哪些设置会被 share / queue 使用

### 建议输出

- Settings key 表
- Snapshot 字段来源表
- V1 apply -> settings -> snapshot 对照表

### 触发时机

新增默认配置字段、share 默认行为、输出目标字段前。

## 路线 3：Queue / Share Runtime 深挖

### 目标

把分享接单到队列执行的运行时链路拆清楚。

### 建议输出

- Intake request 生命周期
- Queue job / task 状态机
- temporary file cleanup 路线
- diagnostics 记录点

### 触发时机

修改分享进入、后台处理、失败恢复、通知逻辑前。

## 路线 4：Renderer / Export Fidelity 深挖

### 目标

明确 preview、renderer、export、metadata writeback 的一致性边界。

### 建议输出

- Renderer 路由表
- Classic White / Immers White 布局责任表
- export metadata 写回流程图
- snapshot / readback 测试索引增强

### 触发时机

修改最终输出图像、Logo、导出文件、metadata 保留前。

## 路线 5：Memory Engine 深挖

### 目标

把 IA-003 相关路径从“知道有哪些文件”推进到“知道 Subject、Snapshot、Expression、Module 如何接起来”。

### 建议输出

- `MemorySubject` 字段语义表
- `ConfigurationSnapshot` 字段来源表
- `ProductionMemoryResolver` 调用图
- `MemoryExpressionEngine` 策略说明

### 触发时机

继续 IA-003B / IA-003C / IA-003D 或新增 Memory 表达能力前。

## 路线 6：测试地图深化

### 目标

从“功能到测试文件”推进到“功能到具体测试意图”。

### 建议输出

- 每个测试文件保护什么行为
- 哪些测试是 contract 级
- 哪些测试是 migration 级
- 哪些测试是 renderer / snapshot / readback 级

### 触发时机

准备大范围重构或准备 release checkpoint 前。

## 推荐顺序

如果没有新的具体功能牵引，建议按这个顺序：

1. Settings / Snapshot 深挖
2. Queue / Share Runtime 深挖
3. V1 主界面精读
4. Memory Engine 深挖
5. Renderer / Export Fidelity 深挖
6. 测试地图深化
