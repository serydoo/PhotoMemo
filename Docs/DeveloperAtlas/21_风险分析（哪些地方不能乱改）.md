# 风险分析（哪些地方不能乱改）

## 最高风险原则

不是所有“大文件”都同样危险。

当前真正高风险的，是那些同时满足下面几条的文件：

- 体量大
- 依赖面广
- 跨层
- 处在入口或共享边界上
- 一改就可能影响 V1 与 V2 并存关系

## 风险区 1：V1 总壳

文件：

- [PhotoMemoiOSV1View.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/iOS/Views/PhotoMemoiOSV1View.swift)

风险原因：

- 约 2119 行
- 同时持有 tab、session、本地编辑状态、bootstrap、配置应用、输出设置、若干 coordinator
- 很多新增功能会“自然地想塞进去”

推荐修改方式：

- 优先增加到现有 `V1*Coordinator` 或 `V1*Surface`
- 只有页面装配必须知道的东西才留在壳里

## 风险区 2：系统装配中心

文件：

- [AppEnvironment.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Architecture/AppEnvironment.swift)

风险原因：

- 它装配 services、repositories、coordinators、queue、external intake
- 很容易因为“只是加一个依赖”逐渐变成业务堆放点

推荐修改方式：

- 只在确实需要装配新依赖时改
- 不往这里塞规则判断

## 风险区 3：配置持久化中心

文件：

- [SettingsService.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Services/SettingsService.swift)
- [SettingsRepository.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Repositories/SettingsRepository.swift)

风险原因：

- 保存了很多默认配置和历史兼容语义
- V1 bootstrap 和共享快照都可能受影响

推荐修改方式：

- 先理清是页面态还是持久化态
- 尽量经由 repository / coordinator 入口改，不直接四处写 defaults

## 风险区 4：生产构建桥

文件：

- [RecordCardBuildService.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Services/RecordCardBuildService.swift)

风险原因：

- 它把 metadata、anchor、template、memory payload 组合成最终 card
- 属于从“配置世界”进入“渲染世界”的桥

推荐修改方式：

- 先判断新信息应该属于 metadata、memory payload、template 还是 export description

## 风险区 5：最终输出边界

文件：

- [RecordCardExportService.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Services/RecordCardExportService.swift)
- [PhotoLibraryExportService.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Services/PhotoLibraryExportService.swift)

风险原因：

- 直接影响最终导出结果
- 容易把不该属于 renderer 的布局真相继续塞进去

推荐修改方式：

- 如果是表达内容改动，先查上游 build / memory / template
- 如果是绘制实现改动，再进 export / renderer

## 风险区 6：分享接单边界

文件：

- [ExternalPhotoIntakeCenter.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/App/ExternalPhotoIntakeCenter.swift)
- [ShareCoordinator.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Coordinators/ShareCoordinator.swift)
- [BatchQueueStore.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Services/BatchQueueStore.swift)

风险原因：

- 这些文件直接决定“外部照片怎么进来、怎么入队”
- 会影响系统分享真实行为

推荐修改方式：

- 任何修改前先分清：
  - 接单
  - 去重
  - 默认配置冻结
  - 队列持久化
  - 批处理执行
  具体是哪一段

## 当前不建议随意碰的方向

- 不在 Atlas 工作里顺手改 Renderer 规则
- 不顺手重构 `ConfigurationCenter` 架构
- 不把新产品词汇随意倒回旧 `workspace/import/dashboard` 概念
- 不在未确认 IA-003 边界前，把生产改动扩散到 Export / Share / Metadata

## 判断是否该停一下

如果你准备改一个文件时，同时出现下面任意两条，就值得先停一下：

- 这是入口文件
- 这是共享装配文件
- 这是 800 行以上的大文件
- 这是 V1 与 V2 的边界文件
- 这是持久化与共享快照相关文件
- 这是最终输出文件

这通常意味着它不是“顺手改一下”的地方。
