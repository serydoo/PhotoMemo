# Developer Atlas 阅读指南

## 这套资料是什么

`DeveloperAtlas` 不是一次性分析报告，而是时光记的长期开发资料库。

它的目标不是把所有 Swift 文件逐行解释，而是让后续开发优先回答三个问题：

1. 它是什么
2. 它为什么存在
3. 如果我要新增功能，应该从哪里开始改

## 当前范围

当前首期范围聚焦 `V1 + V1 / Configuration Center / V2 并存边界`。

这样做的原因很直接：

- 当前 iOS 入口已经不是纯 V1 单线
- `PhotoMemoiOSV1View` 仍然是重要产品线
- `ConfigurationCenteriOSView` 和新的 Memory Engine 边界已经开始并存
- 如果不先画清边界，后面的函数级和变量级分析会越来越难读

## 使用原则

- Atlas 以中文为主，必要时保留英文代码术语
- 图优先使用 Mermaid，保证 GitHub 和仓库内都能直接维护
- 任何“推荐修改入口”都必须尽量绑定到真实文件，而不是抽象空话
- 任何“不要乱改”的判断都要尽量给出原因
- Atlas 是导航系统，不是重复代码
