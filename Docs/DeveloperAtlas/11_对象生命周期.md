# 对象生命周期

## 为什么这一章重要

很多工程理解问题，不是“不知道这个类干什么”，而是不知道：

- 它什么时候被创建
- 会活多久
- 谁持有它
- 它是页面态、运行时态，还是生产结果对象

## 1. `PhotoMemoAppRuntime`

文件：

- [PhotoMemoAppRuntime.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/App/PhotoMemoAppRuntime.swift)

### 生命周期

```text
App 启动
-> App / Scene 创建 @StateObject runtime
-> runtime 持有 environment / queue / intake / background status
-> root scene 使用 runtime
```

### 特点

- 属于 app 级长生命周期对象
- 不是单页 ViewModel
- 是系统分享、外部 intake、queue 刷新的重要中转层

## 2. `AppEnvironment`

文件：

- [AppEnvironment.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Architecture/AppEnvironment.swift)

### 生命周期

```text
runtime 初始化
-> AppEnvironment.live(...)
-> 组装 services / repositories / coordinators
-> 在 runtime 生命周期内长期存活
```

### 特点

- 更像依赖容器
- 不是状态对象
- 适合长期持有，不适合频繁重建

## 3. `ConfigurationSession`

文件：

- [ConfigurationSession.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/ConfigurationCenter/ConfigurationSession.swift)

### 生命周期

```text
V1 / Configuration Center 页面创建
-> @StateObject session 初始化
-> 页面交互期间持续存活
-> 页面销毁后结束
```

### 特点

- 属于页面级但不是一次性局部变量
- 承载对象中心状态
- 在 V1 路径和 Configuration Center 路径里都有存在感

## 4. `ExternalPhotoIntakeCenter`

文件：

- [ExternalPhotoIntakeCenter.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/App/ExternalPhotoIntakeCenter.swift)

### 生命周期

```text
AppEnvironment / runtime 持有
-> 接收 share / open-url 输入
-> 暂存 pending requests
-> 被 drain
-> 保持长期存在
```

### 特点

- 属于运行时共享对象
- 不是一次性请求对象
- 用来承接外部来源和内部 queue 之间的时间差

## 5. `BatchQueueStore`

文件：

- [BatchQueueStore.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Services/BatchQueueStore.swift)

### 生命周期

```text
AppEnvironment 创建
-> 恢复 persisted jobs
-> 运行期间持续持有 jobs / processing 状态
-> 可能跨 session 恢复
```

### 特点

- 这是长生命周期生产状态对象
- 不是普通 service
- 它的状态会恢复、持续、继续处理

## 6. `SelectedPhoto`

文件：

- [SelectedPhoto.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Models/SelectedPhoto.swift)

### 生命周期

```text
PhotoImportService 导入
-> 建立 SelectedPhoto
-> 参与 preview / build / export
-> 当前处理结束后通常不长期持久化
```

### 特点

- 属于处理中间对象
- 比 URL 更高层，比最终导出结果更早
- 同时包含 image、metadata、source info

## 7. `RecordCard`

文件：

- [RecordCard.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Models/RecordCard.swift)

### 生命周期

```text
RecordCardBuildService.buildCard(...)
-> 生成 RecordCard
-> 交给 export / renderer
-> 作为最终绘制输入使用
```

### 特点

- 属于“输出前最终组合对象”
- 它不是原始事实，也不是 UI 草稿
- 它是 renderer / export 最重要的输入之一

## 生命周期分层总结

### 长生命周期运行时对象

- `PhotoMemoAppRuntime`
- `AppEnvironment`
- `ExternalPhotoIntakeCenter`
- `BatchQueueStore`

### 页面生命周期对象

- `ConfigurationSession`
- V1 各类页面 `@State`

### 处理中间对象

- `SelectedPhoto`
- `ProductionMemoryPayload`

### 输出组合对象

- `RecordCard`
- `BatchConfigurationSnapshot`

## 最有价值的判断

如果你不知道一个功能该把状态放哪里，先问这个对象属于哪种生命周期。

这比先考虑“放哪个文件夹”更重要。
