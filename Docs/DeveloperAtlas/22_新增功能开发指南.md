# 新增功能开发指南

## 开始前先问 4 个问题

每个新功能开工前，先判断：

1. 它是 V1 产品体验，还是 Configuration Center 对象能力
2. 它改变 UI 状态，还是改变生产 pipeline
3. 它需要持久化吗
4. 它会影响最终导出图像吗

这 4 个问题能快速决定修改范围。

## 场景 1：新增一个 V1 预览模块

### 推荐路径

1. 在 `V1PreviewCompositionModule` 增加模块定义
2. 在 `V1PreviewCompositionEngine` 增加 display value 和 template token
3. 在 module library presenter 中确认分类与显示
4. 补测试覆盖 preview render model

### 重点文件

- [V1PreviewCompositionEngine.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/iOS/Views/V1PreviewCompositionEngine.swift)
- [V1ModuleLibraryPresenter.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/iOS/Views/V1ModuleLibraryPresenter.swift)

### 验证重点

- 插入后预览显示正确
- 保存后的 template token 正确
- reset / bootstrap 不丢失

## 场景 2：新增一个默认配置项

### 推荐路径

1. 明确字段属于 V1 页面态还是持久化默认配置
2. 扩展保存 request / receipt
3. 通过 `ConfigurationCoordinator` 写入
4. 在 `SettingsRepository` 和 `SettingsService` 中补持久化
5. 如生产需要，加入 `BatchConfigurationSnapshot`

### 重点文件

- [V1ConfigurationApplyCoordinator.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/iOS/Views/V1ConfigurationApplyCoordinator.swift)
- [ConfigurationCoordinator.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Coordinators/ConfigurationCoordinator.swift)
- [SettingsRepository.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Repositories/SettingsRepository.swift)
- [SettingsService.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/Services/SettingsService.swift)

### 验证重点

- 保存后重启能恢复
- share / queue 使用的是保存后的 snapshot
- 旧默认值不会崩

## 场景 3：新增一个 Memory Subject 字段

### 推荐路径

1. 判断字段属于 identity、relationship、anchor、behavior 还是 decoration
2. 修改 `MemorySubject`
3. 如需旧数据桥接，修改 `MemorySubjectAdapter`
4. 如需编辑，修改 Configuration Center 对象编辑入口
5. 如需生产使用，追到 `ProductionMemoryResolver`

### 重点文件

- [MemorySubject.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/ConfigurationCenter/Models/MemorySubject.swift)
- [MemorySubjectAdapter.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/MemoryEngine/MemorySubjectAdapter.swift)
- [ProductionMemoryResolver.swift](/Users/rui/Desktop/PhotoMemo/Source/PhotoMemo/PhotoMemo/MemoryEngine/ProductionMemoryResolver.swift)

### 验证重点

- Configuration Center 编辑能保存和恢复
- MemoryExpression 输出不退回空值
- V1 兼容路径不被破坏

## 场景 4：新增一个 metadata 展示项

### 推荐路径

1. 在 `PhotoMetadata` 中确认事实字段
2. 在 `PhotoMetadataReader` 里读取并归一化
3. 在 `MetadataContext` / `CardVariableProvider` 中暴露变量
4. 在 V1 module 或 template item 中使用

## 场景 5：新增一个输出选项

### 推荐路径

1. 判断是相册目标、文件策略，还是 renderer 输出样式
2. 如果是目标选择，优先看 output page / album presenter
3. 如果是最终保存，追到 `PhotoLibraryRepository`
4. 如果是图像输出，追到 `RecordCardExportService`

## 场景 6：新增分享 / 队列能力

### 推荐路径

1. 判断是来源接入、payload 去重、queue 状态，还是处理执行
2. 来源接入看 `ExternalPhotoIntakeCenter`
3. 请求处理看 `ShareCoordinator`
4. 队列状态看 `BatchQueueStore`
5. 生产执行看 `BatchProcessingCoordinator`

### 重点文件

## 推荐收尾检查

任何新增功能完成前，至少确认：

- 入口是否正确
- 状态归属是否正确
- 是否需要进入 snapshot
- 是否影响 share extension
- 是否影响 preview / export 一致性
- 是否触碰 Renderer / Metadata / Export 等受控边界
