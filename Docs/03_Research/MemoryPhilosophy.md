# Memory Philosophy

Last updated: 2026-06-22

## Core Idea

EXIF records when a photo was taken.

Memory Engine records where that photo belongs inside a person's life.

MemoMark does not simply display photos. MemoMark presents memories.

## Objective And Emotional Information

Every photo contains objective information:

- capture time
- camera
- lens
- GPS
- exposure values

These answer:

- When?
- Where?
- How?

Every meaningful memory also contains emotional information:

- Life Anchor
- relationship
- family
- child
- travel
- parents
- pets
- anniversaries

These answer:

- What does this moment mean?

MemoMark preserves both.

## Life Position

Life Position is the place a photo occupies inside a person's life timeline.

Examples:

- Baby: 1 year 2 months 16 days old
- Relationship: together for 2865 days
- Marriage: 580 days after wedding
- Pregnancy: 102 days before birth
- Father: 365 days after remembrance
- Travel: 1350 days since first visit

A photo becomes more meaningful when it gains Life Position.

## Memory Timeline

Every Life Anchor creates a timeline.

A photo can belong to multiple timelines simultaneously.

For example, one photo can be:

- part of a child's growth timeline
- part of a marriage timeline
- part of a family travel timeline

Memory Engine calculates all relevant relationships. Presentation Engine decides which relationships the current template expresses.

## Engine Boundaries

Memory Engine does not write stories.

Memory Engine only calculates relationships.

Presentation Engine expresses relationships.

Layout Engine decides how that meaning is presented.

Renderer simply draws.

## Principle

Photos have timestamps.

Memories have positions.

MemoMark exists to preserve the meaning of time.
