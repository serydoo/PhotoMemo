# PhotoMemo UX Principles

1. Photo first. Configuration second.

2. The primary entry is Apple Photos Share Extension.

3. The Main App is a configuration center.

4. The user should accomplish the task with minimum decisions.

5. One screen should solve one problem.

6. Preview is the truth. Avoid duplicate information.

7. Never expose implementation concepts.

Users should never need to understand:

- Template Engine
- Workspace
- Configuration Slot
- Metadata Pipeline

Only show concepts users actually care about.

8. Defaults should be smart.

Configuration should be optional whenever possible.

9. Reduce scrolling. Reduce clicks. Reduce reading.

10. If a feature needs documentation, the UI should probably be redesigned.
