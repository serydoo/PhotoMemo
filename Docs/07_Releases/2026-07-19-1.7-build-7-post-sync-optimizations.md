# MemoMark 1.7 (7) 后续优化说明

日期：2026 年 7 月 19 日

上次 GitHub 同步基线：`f0fdb17b`

统一版本标识：`1.7 (7)`

## 更新概览

本轮在上次 GitHub 同步后继续收口首页与记忆对象编辑体验。重点是减少资料区中过重的自定义卡片感，增加当前 TestFlight 阶段需要的首页反馈入口，并统一发布文档中的版本表达。产品仍保持本地优先，照片处理、渲染和 Apple Photos 工作流没有变化。

## 1. 记忆对象资料区原生视觉收口

- 移除对象名称、昵称、与我的关系、专属称呼四行资料外层的白色圆角卡片、描边和裁切。
- 四项资料现在直接作为对象编辑卡片中的连续原生分组行，降低重复边界和视觉重量。
- “表达称呼”保留整行选择和蓝色语义，背景统一使用配置中心既有选中状态颜色，不再叠加单独的强调色描边。
- 保留原有编辑焦点、键盘操作、表达来源、头像、保存和持久化行为。
- 本轮以用户提供的当前效果图作为现状证据，不将其误作目标参考图，也不重新设计已冻结的 IA-002 Configuration Center 架构。

## 2. 首页意见反馈

- 首页“我的配置”后新增固定的“意见反馈”模块，默认展开并允许用户折叠。
- 折叠状态保存在本机；关闭 App 后仍会保留用户上次选择。
- 模块提示用户在小红书、抖音搜索 `MemoMark`，可联系开发者本人。
- 增加 QQ 交流群 `955680366`，用于交流使用体验、反馈问题和提出定制意见。
- TestFlight 系统内置反馈继续并存，设置页已有的 TestFlight、邮件和 GitHub Issues 渠道保持不变。
- 反馈模块使用独立组件且首页只有一个调用点，当前测试阶段结束后可以单独移除，不影响首页其他功能。

## 3. 发布文档与工作纪律

- 将此前混用的 GitHub 文档标签和设备包描述统一为当前实际产品版本 `1.7 (7)`。
- 修订 Live Photo 记录：主 App 选择器与分享扩展流程均已在签名真机验证中成功，后续工作是扩大生产证据覆盖，而不是补齐不可用流程。
- 增加 UI 修改记录纪律：视觉优化先完整记录现状、目标、范围和验证计划，再集中完成一个有边界的 UI pass。
- 当前截图默认只作为现状证据；只有被明确指定为参考设计时，才用于视觉复刻。

## 验证

- 记忆对象资料区结构与选中状态契约测试通过。
- 首页反馈模块完成红灯到绿灯验证，覆盖单一入口、折叠持久化、渠道文案、辅助功能、窄屏换行约束和 TestFlight 共存。
- `PhotoMemoiOS` iPhone7 arm64 签名构建通过。
- 最新开发包已覆盖安装并成功启动，未卸载 App，也未清除既有本地数据。
- 用户已完成记忆对象资料区真机查看，并确认整体效果较修改前明显改善。

## 同步范围

本次同步包含记忆对象资料区视觉收口、首页意见反馈、对应测试、实施记录和发布文档整理。

BrandMark 机制、资源库和集成方案继续作为本地研究内容保留，不进入本次产品代码和 GitHub 同步范围。

## 保持不变

- 照片继续完全在本机处理，不上传用户照片。
- 不修改原图，只生成新的输出图片。
- 不修改 Renderer、Metadata、Export、Share Extension、Photo Library 或 Layout Engine 行为。
