# MemoMark 1.7 (7) 配置连续性与记忆对象界面收口

日期：2026 年 7 月 21 日

上次 GitHub 同步基线：`f2b50833`

统一版本标识：`1.7 (7)`

## 更新概览

本轮整理上次 GitHub 同步之后完成的配置连续性、完整写入、记忆对象界面和仓库清理工作。重点是让不同记忆对象各自恢复自己的配置和时间锚点，让 Apple Photos 图片说明保留完整的智能模块与自定义内容，并把记忆对象页面统一到配置中心的卡片规范。

产品继续保持本地优先。照片不会上传，原始照片不会被修改，处理结果仍以新文件写回 Apple Photos。

## 用户可感知优化

### 1. 切换记忆对象立即恢复对应配置

- 每个记忆对象继续拥有自己的默认配置和时间锚点。
- 切换对象后立即恢复该对象的完整配置、选中锚点和编辑草稿。
- 不再需要重新点击“保存当前配置”才能让配置生效。
- 活动对象和活动配置指针会独立持久化，不会重新写入或覆盖配置内容。

### 2. 写入与保留恢复完整内容

- “自定义内容”改为补充智能模块结果，而不是替换智能模块结果。
- 实际写入顺序统一为：智能模块结果、自定义补充内容。
- Apple Photos 图片说明改为读取完整 Memory 区域结果。
- Memory 区域中的前置文字、智能模块和后置文字都会按配置顺序保留。

### 3. 记忆来源支持稳定折叠

- 配置中心“记忆来源”标题右侧新增展开/收起操作。
- 用户手动收起后，只要没有切换记忆对象，就持续保持收起。
- 切换记忆对象时自动展开，便于确认新的对象、时间锚点和记忆显示方式。
- 折叠操作仅影响界面展示，不会把配置标记为未保存。

### 4. 记忆对象界面统一到配置中心规范

- 页面左右边界统一为 18pt。
- 外层卡片统一使用 14pt 内距和 18pt 圆角。
- 基本资料采用一层外卡加一层内面板，不再叠加重复卡片。
- 对象名称、表达主体和头像重新平衡布局，减少无效留白。
- 基础资料和时间锚点使用统一的语义图标与颜色。
- “时间锚点”入口更名为“时间锚点配置”，状态显示为“已设定锚点数 N 个”。
- 长字段在卡片内部压缩、截断或换行，不再反向撑宽页面。

## 工程与仓库整理

### 1. 退役旧 MainView 编辑链路

- 删除已经没有运行时调用的 `Views/Main/MainView*`、Workspace、Composer 和旧导入界面。
- macOS 继续使用 `ConfigurationCenterView`。
- iOS 继续使用 `PhotoMemoiOSV1View`。
- 仍然有效的文件表示解析逻辑已迁移到正式 Service 层。
- 删除只服务于旧编辑器的测试和源码契约。

### 2. 建立可复用 UI 规范

- 新增配置中心窗口、页面列、外卡、内面板和行级内容规范。
- 新增 iOS 语义图标储备，统一记忆来源、对象资料、时间锚点、输出和任务相关图标语言。
- 记录 Expression Style System 的未来研究种子，但不启动 V4 实现，不修改 Renderer 或 Layout Engine。

### 3. V3 可靠性审计

- 完成 V3 Production Reliability Certification 审计。
- 当前架构边界保持稳定。
- 后续工程优先级明确为 `TX-001 Export Commit Protocol`，随后是 `BP-001 Execution Budget Enforcement`。
- 本轮不改变 PhotoKit 事务、Renderer、Metadata、Export、Share Extension、Photo Library 或 Layout Engine 行为。
