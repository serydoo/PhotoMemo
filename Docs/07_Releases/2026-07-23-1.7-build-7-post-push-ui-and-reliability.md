# MemoMark 1.7 (7) 后续优化说明

日期：2026 年 7 月 23 日

上次 GitHub 同步基线：`0b932f34`

统一版本标识：`1.7 (7)`

## 更新概览

本轮在上次 GitHub 同步后，继续完成主程序视觉统一、Share Extension 视觉收口，以及批处理、Share handoff、持久化、取消和 PhotoKit 保存链路的风险加固。配置中心仍是主程序的视觉基准，产品继续保持本地优先，照片处理、原图保留和 Apple Photos 工作流原则不变。

## 1. 主程序视觉统一

- 将配置中心已有的内容边界、外层卡片、内层面板、圆角、间距和信息行节奏延伸到 Memory Subject、首页摘要、输出、任务、设置、欢迎和相关导航界面。
- 统一主程序中重复出现的卡片层级，减少同一页面内多重边框、重复容器和局部视觉规则漂移。
- 增加共享设计令牌，集中管理 MemoMark 的颜色、排版角色、页面边距、卡片和内层面板指标。
- 统一语义图标目录，配置、对象、时间锚点、输出、任务、帮助等标题优先使用稳定的 SF Symbols 和语义颜色。
- 保留 Configuration Center 的 `Library -> Interactive Memory Card -> Object Inspector` 架构，不重新引入旧 MainView、Workspace 或批处理工作台概念。

## 2. Share Extension 视觉和层级收口

- 将 Share 确认页整理为 `本次分享` 与 `后台处理` 两个功能模块，减少不必要的装饰性容器。
- 统一摘要行、承诺行、品牌说明和主操作的层级、间距、按钮安全区和卡片几何。
- Share Extension 使用独立的 `MemoMark Share Design v1` 合约，与主程序共享语义和排版原则，但保留紧凑半屏场景所需的独立布局。
- 补充 Dynamic Type、辅助功能标签、窄屏换行和 SF Symbols 语义图标契约。
- 保持本地处理、后台处理和返回 Apple Photos 的产品承诺，不新增网络上传行为。

## 3. 批处理、并发和持久化风险加固

- 队列解码失败和队列写入失败不再静默转换为空队列；启动会暴露错误并停止自动恢复，避免历史任务看似消失。
- UserDefaults 队列保存增加同步和读回校验，BatchQueueStore 不再忽略持久化失败，并提供阻断和重试路径。
- Share 请求的共享 UserDefaults 读改写使用进程内锁和跨进程共享文件锁，避免 Share Extension 与主 App 相互覆盖请求。
- Share 请求改为入队成功或明确终止丢弃后才按请求 ID 确认删除；确认失败会留下诊断信息。
- 处理中的源文件不再被取消操作立即删除；排队任务可以立即清理，正在读取、渲染、导出或保存的任务在释放使用权后再清理。
- 静态图片和 Live Photo 保存携带稳定的 BatchTask UUID，PhotoKit 保存前进行资源查重，并通过同进程保存门串行化查重与保存。
- 在外部副作用前后重新检查取消状态，避免已取消任务被本地状态错误覆盖为完成。

## 4. 测试、诊断和发布验证

- 增加队列损坏、队列写入失败、读回不一致、Share 锁失败、请求确认、并发保留、取消清理和 PhotoKit 幂等身份相关回归覆盖。
- 通过无签名 macOS `PhotoMemo`、iOS `PhotoMemoiOS` 和 `PhotoMemoShareExtension` Debug 构建。
- 通过签名真机构建，版本为 `1.7 (7)`；已覆盖安装到自定义名称为 `iPhone7`、实际型号为 iPhone 17 Pro Max 的设备，未卸载 App 或清除本地数据，并成功启动。
- `git diff --check` 通过。
- 仓库已有完整串行测试证据为 `1,005` 项通过、`1` 项跳过、`0` 项失败；本轮重新执行时 Xcode beta 测试服务卡在 `waiting for workers to materialize`，未观察到业务断言失败，因此该次运行不作为新的完整通过证据。

## 仍待完成

- PhotoKit 已具备任务级幂等查重和同进程串行保存，但严格的“外部 PhotoKit 提交 + 本地提交记录 + 崩溃恢复协调”协议仍需后续 TX-001 完成。
- 取消发生在 Live Photo 外部保存已经开始之后时，PhotoKit 写入本身无法被本地状态立即撤销；仍需进一步定义外部副作用的最终一致性和清理策略。
- 48MP、RAW/ProRAW 等高成本媒体的内存预算目前主要用于分类和诊断，尚未完成 BP-001 级别的调度与峰值强制验证。
- 真机 Apple Photos Share 全流程、强制终止保存窗口和高成本媒体 Instruments/Jetsam 证据仍属于后续生产认证范围。

## 保持不变

- 照片继续完全在本机处理，不上传用户照片。
- 不修改原图，只生成新的输出图片。
- Apple Photos 仍是照片来源、查看和长期管理空间。
- 主程序仍以 Configuration Center 为核心，而不是批处理工作台。
