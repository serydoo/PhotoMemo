# PhotoMemo Privacy Policy

Last updated: July 7, 2026

PhotoMemo is a local-first memory presentation app for Apple Photos.

## Summary

- PhotoMemo does not require an account.
- PhotoMemo does not upload photos to a server.
- PhotoMemo does not modify original photos.
- PhotoMemo generates a new image as output.
- PhotoMemo does not track users.
- PhotoMemo does not use third-party advertising SDKs.
- PhotoMemo does not declare collected data types in its privacy manifests.

## Photo Access

PhotoMemo requests Photo Library access so users can share selected photos from
Apple Photos to PhotoMemo, process those selected photos locally, and save
generated memory-card images back to Apple Photos.

Original photos are not destructively edited. PhotoMemo creates a new generated
image as the output.

## Local Storage

PhotoMemo stores configuration, processing state, and shared workflow data
locally on the device using the app container and App Group container.

Examples include selected Presets, Memory Subject configuration, queue state,
processing state, and local duplicate-detection identifiers for shared items.

## Networking

Normal PhotoMemo operation does not require server communication. The current
app is built around local processing and the Apple Photos share workflow.
