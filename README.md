# 为什么开发 PhotoMemo

PhotoMemo 的诞生并不是因为摄影参数展示，而是源于一个非常真实的家庭需求。

随着孩子出生后，手机和相机里积累了越来越多的照片。

从出生到学会翻身、爬行、走路，再到第一次去公园、第一次去海边、第一次上幼儿园，每个阶段都会拍摄大量照片。

这些照片都被很好地保存在苹果照片（Photos）中，但随着时间推移，经常会遇到这样的问题：

* 这张照片是哪一天拍摄的？
* 当时孩子多大了？
* 是第一次去哪里？
* 那时候使用的是什么设备拍摄的？
* 当时的天气、地点是什么？
* 这段回忆发生在什么时候？

虽然这些信息实际上已经存在于照片的 EXIF 元数据中，但在日常浏览、分享和回忆时，往往无法一眼看到。

很多时候需要：

1. 打开照片详情；
2. 查看拍摄日期；
3. 手动计算孩子年龄；
4. 再结合记忆推断当时的场景。

过程并不直观。

因此产生了一个想法：

**能否在不破坏原始照片的前提下，将这些重要信息直接展示在照片上？**

例如：

```text
iPhone 17 Pro Max

24mm f/1.78

1/120 ISO100

2026.06.16

沐沐
1岁2个月15天
```

当再次翻看照片时，无需打开详情页面，就能立即知道：

* 拍摄时间
* 拍摄设备
* 摄影参数
* 孩子当时的年龄
* 特殊纪念信息

让照片本身成为时间的记录者。

---

# 为什么坚持保留 EXIF

在研究现有解决方案时发现：

很多照片加水印工具都会重新导出图片。

虽然视觉效果正确，但往往会导致：

* GPS 信息丢失
* 拍摄时间丢失
* 镜头信息丢失
* 设备信息丢失

这会破坏苹果照片优秀的管理能力。

对于长期记录家庭生活而言，这是无法接受的。

因此 PhotoMemo 的一个核心原则是：

**生成后的照片必须完整保留原始 EXIF 信息。**

这样在苹果照片中依然能够：

* 按地点浏览照片
* 在地图模式查看旅行轨迹
* 按时间轴查看成长记录
* 使用照片搜索功能
* 查看原始摄影参数

所有系统能力保持不变。

---

# Local First

PhotoMemo 坚持本地优先（Local First）设计理念。

照片属于个人最重要的数据之一。

因此项目希望做到：

* 不上传照片
* 不依赖云端渲染
* 不依赖第三方服务器
* 不依赖在线模板服务

所有处理均在设备本地完成。

用户拥有对自己照片的完全控制权。

---

# 项目的最终目标

PhotoMemo 并不仅仅是一款照片加边框工具。

更希望成为：

**记录时间、地点、成长与回忆的照片伴侣。**

多年以后再次翻开相册时，

不仅能看到照片中的画面，

也能看到那一天的时间、地点、设备，以及那个阶段的人生故事。


PhotoMemo 的目标是：

**完全本地运行、完全离线处理、完全可自定义。**

## 核心功能

### 1. 自动读取照片 EXIF

支持读取：

* 设备型号
* 镜头信息
* 等效焦距
* 光圈
* 快门速度
* ISO
* 拍摄时间
* GPS 坐标

示例：

```text
iPhone 17 Pro Max

24mm f/1.78

1/120 ISO100

2026.06.16 18:35
```

---

### 2. 自动生成底部信息边框

在不裁剪原图的情况下：

```text
原图
+
底部信息栏
=
最终照片
```

支持：

* 白色边框
* 米白边框
* 浅灰边框
* 深色边框

---

### 3. 四区域布局系统

默认布局：

```text
┌──────────────────────────┐
│设备型号        日期时间  │
│                          │
│镜头参数        纪念信息  │
└──────────────────────────┘
```

对应：

* 左上区域
* 左下区域
* 右上区域
* 右下区域

四个独立内容模块。

---

### 4. 模板变量系统

支持变量：

```text
{{model}}

{{focal}}

{{aperture}}

{{shutter}}

{{iso}}

{{date}}

{{time}}

{{location}}

{{age}}
```

示例：

```text
{{model}}
```

自动解析为：

```text
iPhone 17 Pro Max
```

---

### 5. 纪念日系统

支持自定义重要日期：

例如：

```text
沐沐生日

2025-04-20
```

自动计算：

```text
1岁1个月26天
```

或者：

```text
已来到这个世界422天
```

支持：

* 宝宝成长记录
* 结婚纪念日
* 买车纪念日
* 买房纪念日
* 自定义纪念事件

---

### 6. EXIF 信息保留

生成新照片后：

自动复制原图 EXIF 数据。

保留：

* 拍摄时间
* GPS 信息
* 设备型号
* 镜头信息
* 摄影参数

确保照片管理软件仍能正常识别原始拍摄信息。

---

## 设计原则

### Local First

所有照片处理均在本地完成：

* 不上传照片
* 不依赖服务器
* 不依赖云端渲染
* 不依赖第三方图床

---

### Privacy First

用户照片永不离开设备。

---

### Template Driven

所有显示内容均由模板驱动。

支持未来扩展：

* 摄影模板
* 宝宝成长模板
* 旅行模板
* Leica 风格
* Fuji 风格
* Apple 极简风格
